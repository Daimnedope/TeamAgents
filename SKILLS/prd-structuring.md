# Skill: PRD Structuring

## Purpose

Produce a complete PRD with testable acceptance criteria.

## Inputs

- Problem statement
- User stories or user needs
- Scope constraints

## Output

- Filled `docs/prd/PRD.md` sections
- Acceptance criteria checklist
- Risks and assumptions list

## Guardrails

- Every feature must map to at least one acceptance criterion.
- Criteria must be testable and observable.
- Mark out-of-scope items explicitly.

## Reusable Prompt

```
Create a PRD for this feature using our template.
Return in sections: summary, problem, scope, stories, acceptance criteria, risks, milestones.
```
