# Skill: MVP Scoping

## Purpose

Cut non-critical work and keep only high-impact MVP scope.

## Inputs

- PRD draft
- Deadline and team limits
- Technical constraints

## Output

- Prioritized feature list: Must / Should / Later
- Minimal release slice
- Deferred backlog items

## Guardrails

- Keep MVP demonstrable in one short session.
- Remove features that do not validate core value.
- Preserve at least one metric for success.

## Reusable Prompt

```
Scope this PRD to MVP for a 1-2 week student project.
Return Must/Should/Later and explain each cut in one line.
```
