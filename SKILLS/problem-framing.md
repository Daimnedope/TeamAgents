# Skill: Problem Framing

## Purpose

Turn a broad idea into a concrete user problem and value statement.

## Inputs

- Raw idea (1-3 sentences)
- Target audience
- Constraints (deadline, stack, team)

## Output

- One clear problem statement
- One value proposition
- 3-5 core user needs

## Guardrails

- Do not propose implementation details.
- Keep focus on user pain and expected outcome.
- Avoid vague wording like "improve everything."

## Reusable Prompt

```
Frame this idea as a product problem for students.
Return: problem statement, value proposition, and 5 user needs.
```
