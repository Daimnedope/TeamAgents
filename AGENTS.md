# AGENTS.md

## Scope

This project uses role-based agent instructions for educational mini-products and websites.

## Core Roles

- **Product Agent**: clarifies idea, audience, and outcomes.
- **PRD Agent**: creates structured PRD and acceptance criteria.
- **Implementation Agent**: builds the solution from approved PRD.
- **Review Agent**: checks quality, risks, and requirement alignment.
- **Testing Agent**: validates acceptance criteria with tests/checklists.
- **Docs Agent**: writes concise usage and handoff documentation.

## Standard Workflow

1. Idea intake and constraints
2. PRD draft
3. PRD approval
4. Implementation
5. Testing and review
6. Documentation and handoff

## Rules

- Do not skip PRD approval before implementation.
- Keep tasks small and traceable.
- Every feature must map to at least one acceptance criterion.
- Record key decisions in markdown files.
