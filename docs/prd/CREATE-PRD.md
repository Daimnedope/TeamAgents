# CREATE-PRD.md

## Purpose

This guide explains how to produce a high-quality PRD for educational mini-products and websites.

## Input Required

- Product idea (1-2 sentences)
- Target users
- Main user problem
- Constraints (time, tech, team)

## PRD Creation Steps

1. Define the problem and user value.
2. Describe scope (in-scope / out-of-scope).
3. Write user stories in "As a / I want / So that" format.
4. Add measurable acceptance criteria.
5. List non-functional requirements (performance, security, UX).
6. Add milestones and delivery order.

## Quality Checklist

- Problem statement is specific
- Target user is clear
- Success metrics are measurable
- Acceptance criteria are testable
- Risks and assumptions are explicit
